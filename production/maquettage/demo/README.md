# Démo de faisabilité

Les mots d'ordre de cette étape sont :
- Recherche
- Tests

La démo de faisabilité consiste à produire des premières versions d'une multitude d'aspects du projet. En bref, il faut tester le plus d'éléments, mais sans trop approfondir.

L'objectif de la démo est de valider :
- les choix techniques 
- les choix esthétiques visuels et sonores
- les hypothèses d’usage.

Voici des exemples de tâches à effectuer pour une démo de faisabilité :
- Valider les choix esthétiques visuels
    - Concevoir des éléments visuels (assets, interface utilisateur, arrière-plans, animations, etc) qui seront utilisés dans le projet (pas des éléments de présentation ou du site Web). 
    - Concevoir autant les figures que le fond. 
    - Valider leur cohérence entre eux et avec l'esthétique désirée. 
    - Établir la méthode et les outils pour la production des images. 
    - Effectuer un exemple d'intégration dans l'environnement de développement 
- Valider les choix esthétiques sonores
    - Établir la méthode et les outils pour la production des sons. 
    - Concevoir plusieurs sons évènementiels (figure) dictés par le scénario. 
    -  Valider leur cohérence entre eux et avec l'esthétique désirée. 
    - Concevoir une minute d'ambiance sonore (fond) dicté par le concept. 
    - Effectuer un exemple d'intégration dans l'environnement de développement 
- Valider l'environnement de développement
    - Évaluer plusieurs outils ou moteurs (ex. Unity, Max, TouchDesigner, JS, etc.).  
    - Trouver un exemple semblable à ce qui est désiré. 
    - Faire fonctionner cet exemple et démontrer comment il peut être adapté pour les besoins du projet. 
    - Confirmer que l’environnement retenu permet les interactions nécessaires et le rendu visuel souhaité 
- Valider le système de capture/d'interaction : 
    - Identifier les solutions possibles : caméras, capteurs, microphones, Lidar, Arduino, Kinect/Azure, Leap Motion, etc. 
    - Trouver un exemple similaire à ce qui est désiré. 
    - Préparer un exemple permettant d’obtenir des évènements qui peuvent être intégrés à l'environnement de développement.
